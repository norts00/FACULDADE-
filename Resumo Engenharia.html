Resumo Interativo - Engenharia de Software

1. Introdução à Engenharia de Software

A Engenharia de Software é uma disciplina que se concentra no desenvolvimento, manutenção e gerenciamento de softwares de forma sistemática, utilizando princípios da engenharia para garantir qualidade, eficiência e escalabilidade.

Por que a Engenharia de Software é importante?

  • Reduz custos de desenvolvimento e manutenção
  • Aumenta a confiabilidade e segurança dos sistemas
  • Garante que o software atenda aos requisitos do usuário
Qual é o principal objetivo da Engenharia de Software?
Desenvolver software de qualidade de forma sistemática
Criar programas sem seguir metodologias
Focar apenas na velocidade de desenvolvimento
Correto! A Engenharia de Software busca aplicar abordagens sistemáticas para desenvolver software de qualidade.

2. Ciclo de Vida do Software

O ciclo de vida de desenvolvimento de software (SDLC - Software Development Life Cycle) é um processo estruturado que define as fases do desenvolvimento de um software.

Principais Fases:

  1. Levantamento de Requisitos: Coleta das necessidades do cliente.
  2. Análise e Planejamento: Definição dos requisitos técnicos e de negócios.
  3. Projeto (Design): Arquitetura do sistema e interfaces gráficas.
  4. Implementação: Codificação e programação do software.
  5. Testes: Validação e verificação para encontrar falhas.
  6. Implantarão: Entrega do software ao usuário final.
  7. Manutenção: Correção de erros e atualizações.
Exemplo Prático:

Imagine que uma empresa deseja criar um aplicativo de entrega de comida. O ciclo de vida incluiria desde a definição dos requisitos (como rastreamento de pedidos e pagamento online) até a implementação, testes e manutenção contínua.

3. Modelos de Desenvolvimento

Existem diferentes abordagens para o desenvolvimento de software.

Modelo Características Melhor Uso
Cascata (Waterfall) Processo linear e sequencial Projetos com requisitos bem definidos
Espiral Enfatiza prototipagem e avaliação de riscos Projetos com alto risco
Incremental O software é desenvolvido em partes Quando partes do sistema podem ser entregues separadamente
Ágil Desenvolvimento iterativo e colaborativo Ambientes dinâmicos com mudanças frequentes
Exemplo Prático:

Se uma startup precisa lançar rapidamente um MVP (produto mínimo viável), o modelo ágil é a melhor escolha, pois permite feedback constante dos usuários e ajustes rápidos.

4. Qualidade de Software

A qualidade do software envolve a capacidade do sistema de atender aos requisitos e expectativas do usuário.

Principais Características:

  • Confiabilidade: O software executa corretamente sob condições específicas
  • Manutenibilidade: Facilidade de corrigir e melhorar o software
  • Eficiência: Uso otimizado de recursos do sistema
  • Usabilidade: Facilidade de uso e aprendizado
  • Portabilidade: Capacidade de ser executado em diferentes ambientes
Exemplo Prático:

Se um aplicativo de banco online tiver falhas constantes, a confiabilidade estará comprometida, o que pode afetar a reputação do banco e afastar clientes.

5. Métodos de Teste de Software

Os testes garantem que o software funcione corretamente antes de ser lançado.

Tipo de Teste Objetivo Nível
Teste Unitário Verifica partes individuais do código Baixo
Teste de Integração Garante que os módulos funcionem bem juntos Médio
Teste de Sistema Avalia o software como um todo Alto
Teste de Aceitação Verifica se o software atende aos requisitos do cliente Alto
Exemplo Prático:

Antes de lançar um e-commerce, a equipe testa o processo de compra para garantir que os pedidos sejam processados corretamente.

Teste de Integração
Teste de Sistema
Teste de Aceitação
Correto! O Teste de Integração verifica a comunicação e interação entre diferentes módulos do sistema.

6. Engenharia de Requisitos

A engenharia de requisitos define as necessidades do sistema e como ele deve funcionar.

Tipos de Requisitos:

  • Funcionais: O que o sistema deve fazer (ex: cadastrar usuários)
  • Não Funcionais: Qualidade e desempenho (ex: tempo de resposta)
  • Regras de Negócio: Restrições específicas da empresa (ex: políticas de desconto)

Técnicas de Elicitação:

  • Entrevistas com stakeholders
  • Workshops com usuários finais
  • Análise de documentos
  • Prototipação
  • Observação do ambiente de trabalho
Exemplo Prático:

Um sistema de hospital deve permitir o cadastro de pacientes (requisito funcional) e operar com alta disponibilidade (requisito não funcional).
